배경

시중 환율 정보는 수 분~수십 분 지연되며, 단순 통계 기반 예측에 머물러 글로벌 시장 변수를 실시간으로 반영하지 못한다는 한계가 있었다. 이를 해결하기 위해 BTC, JPY, WTI, GOLD, DXY 등 글로벌 금융자산 데이터를 실시간으로 수집하고 ML 모델로 환율 방향을 예측하는 MLOps 시스템을 구축하였다.

본 프로젝트에서 데이터 수집·전처리 파이프라인 설계 및 PostgreSQL 데이터 저장소 구축을 담당하였다. 1분봉 단위 실시간 수집부터 UTC 정렬, 결측 보간, 비활성 구간 제외까지 전처리 전 과정을 구현하였다.

시스템 구조

데이터 수집
yfinance API로 USD/KRW, BTC, JPY, WTI, GOLD, DXY 6종 자산 1분봉 실시간 수집
전처리 파이프라인
UTC timestamp 정렬, Rolling Window 기반 공통 index 생성, 선형 보간, 비활성 구간 제외
예측 모델
RandomForest Classifier로 15분 후 환율 방향(상승/하락) 예측, Feature Importance 분석
서빙 & 대시보드
FastAPI 예측 API + Dash 실시간 대시보드 (What-if 시뮬레이션, PDP 그래프 포함)

담당 작업

  • 1
    데이터 수집 파이프라인 설계 — yfinance API를 활용하여 6종 글로벌 금융자산의 1분봉 데이터를 실시간으로 수집하는 파이프라인을 설계하였다. 미국 동부시간 기준 최근 24시간 데이터를 자동으로 수집하도록 구현하였다.
  • 2
    전처리 파이프라인 구현 — UTC timestamp 정렬, Rolling Window 기반 공통 1분 index 생성, 주말·공휴일 비활성 구간 제외, 소규모 갭 선형 보간 등 전처리 전 과정을 src/preprocess.py로 구현하였다.
  • 3
    데이터 저장소 구축 — 전처리된 데이터를 CSV 및 PostgreSQL에 자동 적재하는 파이프라인을 구축하였다. 실행 시 수집·전처리·저장이 일괄 자동화되도록 설계하였다.
  • 4
    GitHub 협업 환경 구성 — 팀 협업을 위한 GitHub 레포지토리를 구성하고 버전관리 환경을 구축하였다. 예측 모델 및 대시보드 개발에도 참여하였다.

시사점

분석에서 파이프라인까지

단순 분석을 넘어 데이터 수집부터 저장까지 자동화된 파이프라인을 직접 설계·구현하였다. 실시간 금융 데이터의 특성(시간대 불일치, 비활성 구간, 결측 패턴)을 이해하고 이를 처리하는 전처리 로직을 설계하는 경험을 쌓았다.

금융 도메인 데이터 이해

글로벌 금융자산 간의 연동 관계(DXY·GOLD·WTI·BTC·JPY)를 이해하고, 이를 환율 예측 모델의 입력 변수로 활용하는 과정에서 금융 시장 데이터의 특성을 직접 경험하였다.
